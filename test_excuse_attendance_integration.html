üéì Excuse Letter + Attendance Integration Test

How it works: When a teacher takes attendance, the system automatically checks if any students have approved excuse letters for that date and class. If a student is marked "absent" but has an approved excuse letter, they are automatically marked as "excused" instead.

üìã Complete Flow Example

Step 1: Student Submits Excuse Letter

Student submits an excuse letter for 2025-08-02:

POST {{base_url}}/api/excuse-letters/submit Authorization: Bearer [STUDENT_TOKEN] { "class_id": "4", "date_absent": "2025-08-02", "reason": "Medical appointment - had to visit the doctor" }

Step 2: Teacher Reviews and Approves

Teacher approves the excuse letter:

PUT {{base_url}}/api/excuse-letters/update/1 Authorization: Bearer [TEACHER_TOKEN] { "status": "approved", "notes": "Approved - medical appointment is valid reason" }

Step 3: Teacher Takes Attendance

Teacher marks student as "absent" for 2025-08-02:

POST {{base_url}}/api/attendance/record Authorization: Bearer [TEACHER_TOKEN] { "student_id": "STU123456789", "class_id": "4", "date": "2025-08-02", "status": "absent" }

‚úÖ Automatic Result:

The system automatically detects the approved excuse letter and changes the status from "absent" to "excused" with a note:

{ "status": true, "message": "Attendance recorded successfully", "data": { "attendance_id": "15", "student_id": "STU123456789", "class_id": "4", "date": "2025-08-02", "status": "excused", // ‚Üê Changed from "absent" to "excused" "notes": "Auto-excused: Medical appointment - had to visit the doctor", "student_name": "John Doe", "excuse_letter": { "letter_id": "1", "reason": "Medical appointment - had to visit the doctor", "status": "approved", "teacher_notes": "Approved - medical appointment is valid reason" } } }

üîç New Endpoints for Teachers

Get Students with Excuse Letter Status

Before taking attendance, teachers can check which students have approved excuse letters:

GET {{base_url}}/api/attendance/students/4/2025-08-02 Authorization: Bearer [TEACHER_TOKEN]

Response shows students with their excuse letter status:

{ "status": true, "message": "Students retrieved successfully", "data": { "class": { ... }, "students": [ { "user_id": "STU123456789", "full_name": "John Doe", "student_num": "2024-0001", "email": "john@example.com", "excuse_letter": { "letter_id": "1", "reason": "Medical appointment - had to visit the doctor", "status": "approved", "teacher_notes": "Approved - medical appointment is valid reason" } }, { "user_id": "STU987654321", "full_name": "Jane Smith", "student_num": "2024-0002", "email": "jane@example.com", "excuse_letter": null // No excuse letter } ], "date": "2025-08-02" } }

‚ö° Bulk Attendance with Auto-Excuse

Bulk Attendance Recording

When recording multiple students at once, the system automatically applies excuse letters:

POST {{base_url}}/api/attendance/bulk-record Authorization: Bearer [TEACHER_TOKEN] { "class_id": "4", "date": "2025-08-02", "attendance_records": [ { "student_id": "STU123456789", "status": "absent" // Will become "excused" automatically }, { "student_id": "STU987654321", "status": "present" // Stays "present" (excuse doesn't override present) }, { "student_id": "STU555666777", "status": "absent" // Stays "absent" (no excuse letter) } ] }

üìä Attendance Status Types

  • present - Student is present in class
  • absent - Student is not present (no excuse)
  • excused - Student is absent but has approved excuse letter
  • late - Student arrived late

üéØ Key Features

‚úÖ Automatic Excuse Application

  • When teacher marks student as "absent", system checks for approved excuse letters
  • If approved excuse exists, status automatically changes to "excused"
  • Original excuse reason is added to attendance notes

‚úÖ Present Status Protection

  • If student is marked "present", excuse letters don't override this
  • Present status takes priority over excuse letters

‚úÖ Teacher Visibility

  • Teachers can see excuse letter status before taking attendance
  • Excuse letter details are included in attendance responses
  • Works with both individual and bulk attendance recording

‚úÖ Real-time Integration

  • No manual intervention required
  • Automatic status updates during attendance recording
  • Immediate feedback with excuse letter details
