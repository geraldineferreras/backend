üìã Feature Overview

This feature automatically marks a student's attendance as "Excused" when a teacher approves their excuse letter.

How it works:
  • Student submits an excuse letter for a specific date
  • Teacher reviews and approves the excuse letter
  • System automatically creates or updates attendance record with "Excused" status
  • Student receives notification about the approval

üîÑ Testing Workflow

Step 1: Submit Excuse Letter (Student)

{ "class_id": "11", "date_absent": "2025-08-08", "reason": "Medical appointment - had a doctor's checkup" }

Step 2: Check Current Attendance Status

GET /api/attendance/records/11/2025-08-08

Step 3: Approve Excuse Letter (Teacher)

PUT /api/excuse-letters/update/{letter_id}
{ "status": "approved", "teacher_notes": "Excuse approved. Medical appointments are valid reasons for absence." }

Step 4: Verify Attendance is Marked as Excused

GET /api/attendance/records/11/2025-08-08

üîç What Happens Behind the Scenes

When a teacher approves an excuse letter:
  1. System checks if attendance record exists for that student, class, and date
  2. If attendance exists: Updates status to "Excused" and adds note
  3. If no attendance exists: Creates new attendance record with "Excused" status
  4. Sends notification to student about approval
  5. Logs the action for audit purposes
